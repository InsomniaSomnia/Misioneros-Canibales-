01 Descripción del Problema

¿Qué debemos resolver?

Tres misioneros (M) y tres caníbales (C) deben cruzar un río usando una barca con capacidad máxima de 2 personas. Todos inician en la orilla izquierda. El objetivo es llevarlos a todos a la orilla derecha respetando la restricción crítica de seguridad.

⚠ Restricción clave: En ningún momento puede haber más caníbales que misioneros en cualquiera de las dos orillas (cuando hay al menos un misionero presente). Si eso ocurre, los caníbales se comen a los misioneros → estado inválido.

🚤

La Barca

Capacidad: 1 ó 2 personas. No puede cruzar vacía. Inicia en el lado izquierdo.

⛪

Los Misioneros

3 individuos que deben ser protegidos. Nunca pueden quedar en minoría ante los caníbales.

🗡️

Los Caníbales

3 individuos. Peligrosos si superan en número a los misioneros en alguna orilla.

¿Por qué es útil en IA?

Este problema es un ejemplo perfecto de búsqueda en espacio de estados. Tiene: un estado inicial definido, un estado objetivo definido, operadores (acciones) limitados y restricciones claras. Exactamente el tipo de problema que BFS y DFS están diseñados para resolver.

02 Modelado Formal

Representación del Problema

Antes de escribir código, debemos definir el problema matemáticamente. Esto es lo que el algoritmo "entiende".

Representación del Estado

Un estado es una tupla de 3 valores que captura toda la información necesaria para describir la situación actual:

ESTADO = (M_izq, C_izq, barca)
3 valores · enteros simples
M_izq
Número de misioneros en orilla izquierda  →  [0, 1, 2, 3]
C_izq
Número de caníbales en orilla izquierda  →  [0, 1, 2, 3]
barca
1 = izquierda  |  0 = derecha
M_der (implícito)
3 − M_izq
C_der (implícito)

Operadores — Acciones Posibles

Cada acción envía personas en la barca. Si la barca está en la izquierda, las personas viajan hacia la derecha (se restan de M_izq y C_izq), y viceversa. Hay 5 movimientos posibles por viaje:

Acción Misioneros enviados Caníbales enviados Descripción
A110Un misionero cruza solo
A220Dos misioneros cruzan
A301Un caníbal cruza solo
A402Dos caníbales cruzan
A511Un misionero y un caníbal cruzan

Restricciones de Validación

Un estado se considera válido si cumple todas estas condiciones:

R1

Rango válido de personas

Todos los valores deben estar entre 0 y 3: 0 ≤ M_izq ≤ 3 y 0 ≤ C_izq ≤ 3

R2

Misioneros seguros en orilla izquierda

Si hay misioneros en la izquierda, no pueden ser superados por caníbales: M_izq == 0 OR M_izq ≥ C_izq

R3

Misioneros seguros en orilla derecha

Lo mismo aplica para la derecha (valores implícitos): M_der == 0 OR M_der ≥ C_der

Espacio Total de Estados

Teóricamente hay 4 × 4 × 2 = 32 estados posibles. Al aplicar las restricciones, quedan aproximadamente 16 estados válidos que el algoritmo puede visitar. De esos, solo algunos llevan al objetivo.

05 Algoritmos de Búsqueda

BFS vs DFS — Diferencia Fundamental

Ambos algoritmos recorren el mismo espacio de estados. La diferencia es el orden en que exploran los nodos. Esa diferencia cambia completamente el comportamiento.

🔵 BFS — Breadth First Search

  • Usa una Cola (Queue — FIFO): primero en entrar, primero en salir
  • Explora nivel por nivel: todos los nodos del nivel N antes del nivel N+1
  • Garantiza encontrar el camino más corto (mínimo de pasos)
  • Consume más memoria: guarda todos los nodos del nivel actual
  • Para este problema: encuentra la solución óptima en 11 pasos

🔴 DFS — Depth First Search

  • Usa una Pila (Stack — LIFO): último en entrar, primero en salir
  • Explora una rama hasta el fondo antes de retroceder
  • No garantiza el camino más corto (puede encontrar una solución más larga)
  • Menor uso de memoria que BFS
  • Para este problema: puede variar entre 11 y 13+ pasos según el orden de movimientos

Analogía con el Ejemplo Java

En el código Java de Grafo.java se usa Stack<Nodo> pila → eso es DFS. Para hacer BFS, solo cambiamos la pila por una cola (Queue / LinkedList). La lógica del resto del código es idéntica. En JavaScript: 

// DFS — usa array como PILA (push/pop)
const estructura = [];        // pila
estructura.push(inicio);
const actual = estructura.pop(); // saca del FINAL → LIFO

// BFS — usa array como COLA (push/shift)
const estructura = [];        // cola
estructura.push(inicio);
const actual = estructura.shift(); // saca del INICIO → FIFO

✨ Clave: La única diferencia entre BFS y DFS en el código es .pop() vs .shift(). Todo lo demás —validación, generación de sucesores, detección del objetivo— es exactamente igual.

06 Implementación Completa

Código JavaScript — Módulo a Módulo

📦 Nodo.js — La unidad básica

Equivalente a Nodo.java. Guarda el estado y una referencia al nodo padre para reconstruir el camino al llegar al objetivo. 

// ───────────────────────────────────────────────────────
// Nodo.js — Representa un estado en el espacio de búsqueda
// Equivalente a Nodo.java del ejemplo
// ───────────────────────────────────────────────────────

class Nodo {
  constructor(misioneros, canibales, barca, padre = null, accion = "") {
    this.misioneros = misioneros;  // cuántos M en orilla izquierda
    this.canibales  = canibales;   // cuántos C en orilla izquierda
    this.barca      = barca;       // 1 = izquierda, 0 = derecha
    this.padre      = padre;       // nodo del que venimos (para reconstruir ruta)
    this.accion     = accion;      // descripción del movimiento que generó este nodo
  }

  // Genera un identificador único para detectar estados repetidos
  // Equivalente a getId() en Nodo.java
  getId() {
    return `${this.misioneros},${this.canibales},${this.barca}`;
  }
}

module.exports = Nodo;

🌐 Problema.js — Las reglas del dominio

Equivalente a la lógica de Grafo.java (las reglas del laberinto: dónde hay muros, qué movimientos son posibles). Aquí están las reglas del problema de misioneros y caníbales. 

// ───────────────────────────────────────────────────────
// Problema.js — Reglas y lógica del problema
// Equivalente a Grafo.java (la parte de validación y movimientos)
// ───────────────────────────────────────────────────────

const Nodo = require('./Nodo');

// Los 5 movimientos posibles: [misioneros que cruzan, caníbales que cruzan]
// Equivalente al array de movimientos en Grafo.java
const MOVIMIENTOS = [
  { m: 1, c: 0, desc: "1 misionero cruza"        },
  { m: 2, c: 0, desc: "2 misioneros cruzan"      },
  { m: 0, c: 1, desc: "1 caníbal cruza"          },
  { m: 0, c: 2, desc: "2 caníbales cruzan"       },
  { m: 1, c: 1, desc: "1 misionero + 1 caníbal"  },
];

// Verifica si un estado es válido (no viola restricciones)
// Equivalente al if (laberinto[x][y] != 1) en Grafo.java
function esValido(m, c) {
  // R1: Valores dentro del rango [0, 3]
  if (m < 0 || m > 3 || c < 0 || c > 3) return false;

  // R2: En orilla izquierda, M no puede ser superado por C (si hay M)
  if (m > 0 && m < c) return false;

  // R3: En orilla derecha (3-m y 3-c), mismo principio
  const mDer = 3 - m;
  const cDer = 3 - c;
  if (mDer > 0 && mDer < cDer) return false;

  return true;
}

// Verifica si el nodo es el estado objetivo
function esObjetivo(nodo) {
  return nodo.misioneros === 0 && nodo.canibales === 0 && nodo.barca === 0;
}

// Genera los nodos sucesores (hijos) de un nodo dado
// Equivalente al bloque de "Agregar vecinos" en Grafo.java
function generarSucesores(nodo) {
  const sucesores = [];
  const { misioneros: m, canibales: c, barca: b } = nodo;

  for (const mov of MOVIMIENTOS) {
    let nuevoM, nuevoC;

    if (b === 1) {
      // Barca en izquierda → personas van hacia la derecha (se restan)
      nuevoM = m - mov.m;
      nuevoC = c - mov.c;
    } else {
      // Barca en derecha → personas regresan a izquierda (se suman)
      nuevoM = m + mov.m;
      nuevoC = c + mov.c;
    }

    if (esValido(nuevoM, nuevoC)) {
      sucesores.push(new Nodo(nuevoM, nuevoC, 1 - b, nodo, mov.desc));
    }
  }

  return sucesores;
}

module.exports = { esValido, esObjetivo, generarSucesores };

🔍 Buscador.js — Los algoritmos BFS y DFS

Aquí vive la lógica de búsqueda pura. La estructura es casi idéntica al método busquedaProfundidad de Grafo.java. Para BFS solo cambiamos pop() por shift(). 

// ───────────────────────────────────────────────────────
// Buscador.js — Algoritmos de búsqueda BFS y DFS
// Basado en la estructura de busquedaProfundidad() de Grafo.java
// ───────────────────────────────────────────────────────

const { esObjetivo, generarSucesores } = require('./Problema');

// Reconstruye la ruta desde el nodo objetivo hasta el inicio
// (siguiendo los punteros "padre", igual que en listas enlazadas)
function reconstruirRuta(nodo) {
  const ruta = [];
  let actual = nodo;
  while (actual !== null) {
    ruta.unshift(actual); // agrega al inicio
    actual = actual.padre;
  }
  return ruta;
}

// ══════════════════════════════════════════════════════
// BFS — Búsqueda en Amplitud (usa COLA → shift)
// ══════════════════════════════════════════════════════
function bfs(inicio) {
  const cola      = [inicio];       // equivalente a Queue en Java
  const visitados = new Set();      // equivalente a ArrayList visitados
  let   nodos     = 0;

  while (cola.length > 0) {

    const actual = cola.shift();    // ← FIFO: saca del FRENTE
    nodos++;

    if (visitados.has(actual.getId())) continue;
    visitados.add(actual.getId());

    console.log(`[BFS] Visitando: (${actual.misioneros}M, ${actual.canibales}C, barca:${actual.barca === 1 ? 'izq':'der'})`);

    if (esObjetivo(actual)) {
      return { ruta: reconstruirRuta(actual), nodosExplorados: nodos };
    }

    const hijos = generarSucesores(actual);
    for (const hijo of hijos) {
      if (!visitados.has(hijo.getId())) {
        cola.push(hijo);             // agrega al FINAL de la cola
      }
    }
  }
  return null; // No se encontró solución
}

// ══════════════════════════════════════════════════════
// DFS — Búsqueda en Profundidad (usa PILA → pop)
// Basado directamente en busquedaProfundidad() de Grafo.java
// ══════════════════════════════════════════════════════
function dfs(inicio) {
  const pila      = [inicio];       // equivalente a Stack<Nodo> en Java
  const visitados = new Set();
  let   nodos     = 0;

  while (pila.length > 0) {

    const actual = pila.pop();      // ← LIFO: saca del FINAL
    nodos++;

    if (visitados.has(actual.getId())) continue;
    visitados.add(actual.getId());

    console.log(`[DFS] Visitando: (${actual.misioneros}M, ${actual.canibales}C, barca:${actual.barca === 1 ? 'izq':'der'})`);

    if (esObjetivo(actual)) {
      return { ruta: reconstruirRuta(actual), nodosExplorados: nodos };
    }

    const hijos = generarSucesores(actual);
    for (const hijo of hijos) {
      if (!visitados.has(hijo.getId())) {
        pila.push(hijo);             // agrega al FINAL de la pila
      }
    }
  }
  return null;
}

module.exports = { bfs, dfs };

📄 index.js — El punto de entrada

Equivalente a ProblemaLaberinto.java con el método main. Crea el estado inicial, ejecuta ambos algoritmos e imprime los resultados. 

// ───────────────────────────────────────────────────────
// index.js — Punto de entrada (equivalente al main de Java)
// ───────────────────────────────────────────────────────

const Nodo           = require('./Nodo');
const { bfs, dfs } = require('./Buscador');

// Estado inicial: (3M, 3C, barca en izquierda)
const inicio = new Nodo(3, 3, 1);

function imprimirResultado(nombre, resultado) {
  console.log("\n══════════════════════════════════");
  console.log(`  Resultado ${nombre}`);
  console.log("══════════════════════════════════");

  if (!resultado) {
    console.log("  ✗ No se encontró solución.");
    return;
  }

  const { ruta, nodosExplorados } = resultado;

  console.log(`\n  Estado inicial: (3M, 3C, barca: izquierda)`);
  console.log(`  Estado objetivo: (0M, 0C, barca: derecha)\n`);

  ruta.forEach((nodo, i) => {
    const lado = nodo.barca === 1 ? 'izquierda' : 'derecha';
    if (i === 0) {
      console.log(`  Paso 0: [INICIO] → Izq(${nodo.misioneros}M, ${nodo.canibales}C) | Der(${3-nodo.misioneros}M, ${3-nodo.canibales}C) | Barca: ${lado}`);
    } else {
      console.log(`  Paso ${i}: [${nodo.accion}] → Izq(${nodo.misioneros}M, ${nodo.canibales}C) | Der(${3-nodo.misioneros}M, ${3-nodo.canibales}C) | Barca: ${lado}`);
    }
  });

  console.log(`\n  ✓ Solución encontrada en ${ruta.length - 1} pasos`);
  console.log(`  ✓ Nodos explorados: ${nodosExplorados}`);
}

// Ejecutar BFS
console.log("\n[BFS] Iniciando búsqueda en amplitud...");
const resBFS = bfs(inicio);
imprimirResultado("BFS", resBFS);

// Ejecutar DFS
console.log("\n[DFS] Iniciando búsqueda en profundidad...");
const resDFS = dfs(new Nodo(3, 3, 1));
imprimirResultado("DFS", resDFS);

06

Abrir la terminal dentro de la carpeta del proyecto

Este es el paso más importante. La terminal debe estar ubicada dentro de la carpeta misioneros-canibales, no en otra carpeta.

# Si ya creaste la carpeta desde la terminal, ya estás dentro.
# Si abriste la carpeta en VS Code, usa el atajo:

Ctrl + `     # Abre terminal integrada en VS Code (Windows/Linux)
Cmd  + `     # Abre terminal integrada en VS Code (Mac)

# O manualmente: Terminal → Nueva Terminal desde el menú de VS Code

# Para navegar a la carpeta si estás en otra ubicación:
cd ruta/a/tu/carpeta/misioneros-canibales

# Ejemplo Windows:
cd C:\Users\TuNombre\Desktop\misioneros-canibales

# Ejemplo Mac/Linux:
cd ~/Desktop/misioneros-canibales
07

Ejecutar el programa

Con la terminal dentro de la carpeta, escribe el comando de ejecución:

node index.js

✅ Si todo está bien verás en la consola los pasos del BFS y DFS impresos uno por uno, seguidos del resumen con la solución en 11 pasos.

⚠ Error común — "Cannot find module": Esto significa que la terminal NO está dentro de la carpeta del proyecto. Usa cd para navegar hasta misioneros-canibales y vuelve a intentar.

Resumen rápido — Solo los comandos

# 1. Crear y entrar a la carpeta
mkdir misioneros-canibales

New-Item Nodo.js, Problema.js, Buscador.js, index.js

node index.js
